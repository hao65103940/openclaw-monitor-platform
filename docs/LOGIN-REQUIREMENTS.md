# 登录页面需求分析

## 🎯 项目背景

为 OpenClaw 监控平台添加登录功能，实现用户认证和权限管理。

---

## 📊 需求分析

### 1. 用户角色

| 角色 | 权限 | 说明 |
|------|------|------|
| **管理员** | 全部权限 | 查看/编辑所有 Agent 配置、查看监控数据 |
| **普通用户** | 只读权限 | 查看监控数据，不可编辑配置 |
| **访客** | 受限权限 | 只能查看仪表盘，不可查看敏感数据 |

### 2. 认证方式

| 方式 | 优先级 | 说明 |
|------|--------|------|
| **账号密码** | P0 | 基础认证，本地存储用户信息 |
| **飞书扫码** | P1 | 集成飞书 OAuth（后续） |
| **企业微信** | P1 | 集成企微 OAuth（后续） |
| **Token 认证** | P2 | API 调用使用 |

### 3. 功能需求

#### 核心功能（P0）

- [ ] 登录页面
  - [ ] 账号密码输入
  - [ ] 记住我（7 天免登录）
  - [ ] 忘记密码（可选）
  - [ ] 登录按钮
- [ ] 登出功能
- [ ] 会话管理（Token 过期自动登出）
- [ ] 路由守卫（未登录跳转登录页）

#### 增强功能（P1）

- [ ] 注册页面
- [ ] 密码修改
- [ ] 用户信息管理
- [ ] 登录日志

#### 高级功能（P2）

- [ ] 双因素认证（2FA）
- [ ] 密码强度检测
- [ ] 登录失败次数限制
- [ ] IP 白名单

## 🔐 安全需求

### 密码安全

| 要求 | 实现方式 |
|------|----------|
| 密码加密存储 | bcrypt 哈希 |
| 密码传输加密 | HTTPS + 前端加密 |
| 密码强度 | 最少 8 位，包含大小写 + 数字 |
| 密码过期 | 90 天强制修改（可选） |

### 会话安全

| 要求 | 实现方式 |
|------|----------|
| Token 存储 | HttpOnly Cookie / localStorage |
| Token 有效期 | 7 天（记住我）/ 24 小时（默认） |
| Token 刷新 | 自动刷新机制 |
| 并发登录 | 允许 / 限制（可配置） |

### 防护机制

| 攻击类型 | 防护措施 |
|----------|----------|
| 暴力破解 | 登录失败 5 次锁定 30 分钟 |
| XSS 攻击 | 输入过滤 + HttpOnly Cookie |
| CSRF 攻击 | CSRF Token 验证 |
| 中间人 | HTTPS 强制 |

## 📁 数据结构

### 用户表（users）

```typescript
interface User {
  id: string;           // 用户 ID
  username: string;     // 用户名（唯一）
  password: string;     // bcrypt 哈希
  email?: string;       // 邮箱
  role: 'admin' | 'user' | 'guest';  // 角色
  createdAt: Date;      // 创建时间
  lastLogin?: Date;     // 最后登录时间
  isActive: boolean;    // 是否激活
}
```

### 会话表（sessions）

```typescript
interface Session {
  id: string;           // 会话 ID
  userId: string;       // 用户 ID
  token: string;        // JWT Token
  expiresAt: Date;      // 过期时间
  createdAt: Date;      // 创建时间
  ipAddress?: string;   // 登录 IP
  userAgent?: string;   // 设备信息
}
```
