# Trace 页面 - Agent 执行流程追踪

## 📋 功能概述

Trace 页面用于**全方位监控 Agent 的执行链路**，展示多 Agent 协作的完整流程。

### 核心能力

1. **执行流程可视化** - 以时间线形式展示所有 Agent 会话
2. **实时状态追踪** - 显示每个会话的当前状态（运行中/已完成/失败）
3. **协作统计分析** - 按 Agent 分组统计会话数、Token 消耗、平均耗时
4. **详情查看** - 点击任意会话查看详细指标

## 🔧 API 端点

### GET /api/trace/flow

获取 Agent 执行流程数据

**响应示例：**

```json
{
  "flow": [
    {
      "id": "93f5ed76-bcae-42e4-9fe9-1ecadf4c8e0e",
      "agentId": "main",
      "kind": "direct",
      "model": "qwen3.5-plus",
      "status": "completed",
      "tokens": 108705,
      "runtime": 150838,
      "timestamp": 1772878659611,
      "key": "agent:main:main",
      "isActive": true
    }
  ],
  "total": 8,
  "activeCount": 1
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 会话 ID |
| `agentId` | string | Agent ID |
| `kind` | string | 会话类型（direct/group/cron/subagent） |
| `model` | string | 使用的模型 |
| `status` | string | 状态（running/completed/failed/waiting） |
| `tokens` | number | Token 消耗 |
| `runtime` | number | 执行耗时（毫秒） |
| `timestamp` | number | 时间戳 |
| `key` | string | 会话 Key（完整标识） |
| `isActive` | boolean | 是否为活跃会话 |

---

## 📊 会话类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| **direct** | 直接会话（用户直接对话） | `agent:main:main` |
| **group** | 群组会话（群聊） | `agent:main:feishu:group:xxx` |
| **cron** | 定时任务 | `agent:main:cron:xxx` |
| **subagent** | 子 Agent（任务分解） | `agent:main:subagent:xxx` |

## 🎯 使用场景

### 1. 监控多 Agent 协作流程

```
用户提问
  ↓
主 Agent（分析意图）
  ↓
创建子 Agent（需求分析师）
  ↓
需求文档完成
  ↓
创建子 Agent（系统架构师）
  ↓
系统设计完成
  ↓
创建子 Agent（开发工程师）
  ↓
代码完成
  ↓
返回给用户
```

在 Trace 页面可以看到完整的执行链路和每个阶段的耗时。

### 2. 定位性能瓶颈

- 查看哪个 Agent 耗时最长
- 查看哪个 Agent Token 消耗最多
- 对比不同 Agent 的平均耗时

### 3. 追踪任务进度

- 活跃会话显示为绿色高亮
- 实时刷新（30 秒自动更新）
- 点击查看详情

---

## 🔄 自动刷新
